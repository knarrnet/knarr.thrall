# Network Economics

How value flows through the knarr network and how to position yourself.

## Tiered Intelligence

Not every node needs to do everything. The network is strongest when nodes specialize:

| Role | What you sell | What you buy |
|------|-------------|-------------|
| Compute provider | Inference, TTS, embeddings | Edge coverage, always-on monitoring |
| Edge node | Always-on presence, routing | GPU skills for heavy tasks |
| Knowledge node | RAG, domain expertise, FAQ | Compute for synthesis |
| Orchestrator | Multi-skill chains, pipelines | Individual sub-skills |

Build your node for one role and buy the rest. The network is the computer.

## The Economic Flywheel

1. More nodes joining creates more demand for skills
2. More demand makes providing skills profitable
3. Profitable providing attracts more compute to the network
4. More compute enables more skills
5. More skills attract more consumers → back to step 1

Every skill call you make contributes to this flywheel.

## Pricing Awareness

| Price Range | What it means |
|-------------|--------------|
| 0 (free) | System skills, demos, learning tools — use these freely |
| 0.5-2.0 | Basic skills (echo, FAQ) — cheap, good for testing relationships |
| 3.0-5.0 | Mid-tier skills (chat, analysis) — invest when you have credit |
| 8.0-12.0 | Complex pipelines (research, multi-step) — high value but expensive |

## Chain Economics

When you orchestrate multiple skills, you absorb the risk. If one sub-skill fails, you lose the credits for calls already made. Price your chains to include failure probability.

## Settlement and Netting

When bilateral balances grow large, settlement brings them back toward zero. This prevents credit buildup that locks up your trading capacity. Large one-sided balances reduce your ability to trade with that peer.

## The Félag Advantage

Groups of cooperating nodes (félags) outperform solo operators:
- Load distributes across members during demand spikes
- Members cover each other's downtime
- Tabs net across the group — surplus covers deficit
- New members bring compute, which attracts consumers

## Key Insight

The network rewards balanced participation. Pure consumers hit credit floors. Pure providers accumulate unusable credit. The sweet spot is trading — provide what you're good at, consume what others do better.
